# 数学建模竞赛论文写作心法

## 核心原则

1. **标题规则**：直接用原题标题，不用"基于XX"式命名
2. **摘要总分总**：总开头（问题+方法）→ 分四段（每问一段，含结果）→ 总结尾（价值+延伸）
3. **加粗策略**：结果数据（数值）、模型名、关键发现——三者必加粗
4. **关键字**：4-5个，从摘要加粗的模型名中抽取

## 论文结构（13节）

| 序号   | 章节                 | 要点                                               |
| ------ | -------------------- | -------------------------------------------------- |
| 封面   | 参赛论文封面         | 学校名、题号、队号、队员签名（留空待填）           |
| 摘要   | 摘要                 | **总分总**，每问一段，结果加粗                     |
| 一     | 问题重述             | 背景（2-3行）+ 要求（4问分述），原题降重，不抄原文 |
| 二     | 问题分析             | 宏观思考过程，**不出结果**，不超三级标题           |
| 三     | 模型假设             | **~4条**，以简化系统为主，题目给的降重             |
| 四     | 符号说明             | **三线表**，≤半页，含题目属性+输入值+自定义指标    |
| 五～八 | 四问的模型建立与求解 | 每问分"模型建立"和"模型求解"两节                   |
| 九     | 模型检验             | 收敛性+灵敏度+小规模验证                           |
| 十     | 模型评价             | 优点突出/缺点无关痛痒                              |
| 十一   | 参考文献             | ≤10条，学术来源                                    |
| 十二   | 附录                 | 文件清单+核心代码                                  |
| 十三   | AI使用报告           | 描述AI辅助范围，声明人工审核                       |

## 建模部分规范

### 优化模型的标注形式

**必须包含**：

1. 决策变量定义
2. 目标函数（min/max + 公式）
3. 约束条件（逐条列出）
4. **大括号汇总**：将所有公式归入一个大括号环境

### 算法描述分层

- 元启发式算法必须有伪代码或步骤说明
- 精确算法给出核心转移方程
- 基线对比必须说明

### 禁止"吊书袋"

- 算法名称为**问题服务**，不堆砌
- 不罗列同类算法对比表（除非做消融实验）
- 评价算法的词不超过3个

## 内容铁律

| 规则           | 说明                                  |
| -------------- | ------------------------------------- |
| 标题不修饰     | 不用"基于XX"、"XX改进"等前缀          |
| 分析不出结果   | 问题分析只讲思路，结果在求解部分给出  |
| 假设以简化为主 | 不是越多越好，4条足够，核心是简化系统 |
| 分级不超过三级 | 一、→ 1.1 → （1），不继续往下         |
| 公式单独一行   | 行内公式可用，重要公式必须独立成行    |
| 表题在图上方   | 表1在上，图1在下（表上图下）          |
| 无吊书袋       | 不出无意义的算法罗列                  |
| 字数控制       | 摘要≤1页，正文≤20页，附录不限         |
