# 数学建模全流程分步工作流

完整工作流：**领域理解 → 头脑风暴 → 计划 → 资源调查 → 实现求解 → 简化优化 → 验证确认 → 论文生成 → 自评验收**

---

## Phase 0：问题理解

**目标**：保证人和AI对问题的理解一致。

**你要做的**：

- 把题目原文发给 AI
- 说清楚背景和约束

**AI 做的**：

- 复述问题理解，确认没有歧义
- 判断题型（优化类/评价类/预测类/数据类）
- 评估规模（小 n≤15 / 中 n≤100 / 大 n>100）

**输出**：问题理解确认书（一段话，你点头了再往下）

---

## Phase 1：头脑风暴（Brainstorming）

**目标**：发散思路，不做决定。

**AI 做的**：

- 给出 2-3 种可行技术路线
- 每种路线说明：思路 + 优缺点 + 预期效果
- 不评估哪种更好——那是你的决定

**你做的**：

- 听完所有方案
- 选一个方向，或者组合

**关键**：这个阶段不写代码。只聊天。

---

## Phase 2：制定计划（Plan）

**目标**：确定实现方案，列出步骤。

**AI 做的**：

- 根据你选的方向，出具体计划：

```
计划包含：
├─ 做什么（一句话）
├─ 技术方案（算法选型 + 理由）
├─ 涉及文件清单
├─ 实现步骤（分步，每步可独立验证）
├─ 验证方法（怎么确认每步正确）
└─ 需要你做什么
```

- 算法选型参考 `算法选型指南.md`

**你做的**：

- 审核计划
- 说"行"或者调整

---

## Phase 3：资源调查（找 Skill）

**目标**：不重复造轮子。

**AI 做的**：

- 检查已有 skill 库是否有可复用的
- 检查当前项目代码中是否有可复用的函数
- 报告：找到了什么 / 没找到要新建什么

**你做的**：

- 决定复用还是新建

---

## Phase 4：实现求解（Implement）

**目标**：按计划写代码、跑结果。

**AI 做的**：

- 一次实现一个模块
- 每完成一个模块跑通验证
- 记录结果到 `results/`

**分步模式**（推荐）：

```
Step 4.1: 配置和数据准备
  写 config.py, utils.py → 验证数据正确

Step 4.2: Phase1 求解
  写算法代码 → 跑通 → 记录结果

Step 4.3: Phase2 求解
  ...

Step 4.4: Phase3 求解
  ...

Step 4.5: Phase4 求解
  ...
```

**你做的**：

- 每步完成后确认
- 卡住了告诉 AI 调整

---

## Phase 5：简化优化（Simplify）

**目标**：减少冗余、提高效率。

**AI 做的**（参考 `/simplify` 的四个维度）：

| 维度 | 检查内容               |
| ---- | ---------------------- |
| 复用 | 有没有重复实现已有功能 |
| 简化 | 有没有不必要的复杂逻辑 |
| 效率 | 有没有可优化的性能问题 |
| 深度 | 改动在正确层级吗       |

---

## Phase 6：验证确认（Verify）

**目标**：每一项改动的表面可观察验证。

**AI 做的**（参考 `/verify`）：

1. **找变化**：确认改了什么
2. **找表面**：这个改动的用户界面是什么（CLI输出/图表/文件）
3. **驱动它**：跑最小路径让改动代码执行
4. **推它**：边界情况测试
5. **捕获**：记录所有输出

**验证 ≠ 跑测试**。验证 = 跑真的场景看实际输出。

---

## Phase 7：论文生成

**目标**：按规范生成竞赛论文。

**AI 做的**：

- 参考 `写作心法.md` 规范
- 使用 `src/math_modeling/paper_generator.py`：
  1. 配置 `PaperConfig`（学校名、标题等）
  2. 注入 `results.json`（自动填入求解结果）
  3. 生成 .docx

**你做的**：

- 检查论文内容
- 填写封面信息（队员签名等）

---

## Phase 8：自评验收

**目标**：确认交付物质量。

**AI 做的**：

- 对照 `写作心法.md` 逐项检查：

```
□ 标题：不用"基于XX"——直接用原题
□ 摘要：总分总 + 加粗结果
□ 问题分析：不出结果，只讲思路
□ 模型假设：~4条，以简化系统为主
□ 符号说明：三线表，≤半页
□ 模型格式：决策变量→目标→约束→大括号
□ 无吊书袋
□ 分级不超过三级
□ 表题在图上方
```

- 安全扫描：确认无敏感信息泄露
- 整体评分 + 改进建议

**你做的**：

- 最终审核
- 说"行"或者"改"
